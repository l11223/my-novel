## 金手指设定：满级模拟器

### 核心能力

**【模拟】**
- 可在进入任何怪谈前进行模拟
- 模拟中的死亡不是真正的死亡
- 每次模拟保留全部记忆
- 模拟次数无上限

**【推演】**
- 根据已有信息推演隐藏规则
- 推演准确度与已有信息量正相关
- 可推演怪谈核心位置

**【记录】**
- 自动记录所有遇到的规则
- 可对比规则矛盾之处
- 标记"必守规则"与"陷阱规则"

### 限制与代价

1. **时间限制**：模拟时间与现实1:1流动，模拟100次就是100次的时间
2. **精神消耗**：每次模拟死亡都会承受死亡的痛苦记忆
3. **信息局限**：只能获取主角视角的信息，无法全知
4. **无法外带**：模拟中获得的物品无法带出

### 成长路线

| 阶段 | 解锁能力 | 字数节点 |
|------|----------|----------|
| 初始 | 基础模拟（无限次） | 0万 |
| Lv.2 | 加速模拟（2倍速） | 30万 |
| Lv.3 | 规则标记（自动标记关键规则） | 60万 |
| Lv.4 | 多线模拟（同时模拟多个选择） | 100万 |
| Lv.5 | 深度推演（推演怪谈起源） | 150万 |
| Lv.6 | 规则干涉（小范围修改规则） | 200万 |
| Lv.7 | 时间回溯（现实中一次性复活） | 250万 |
| 满级 | 规则重构（重写怪谈规则） | 280万 |

## 写作要点

### 1. 怪谈设计原则
- 每个怪谈都要有逻辑自洽的规则体系
- 必须有"假规则"或"隐藏规则"制造反转
- 规则背后要有故事（怪谈的起源）
- 让读者也能参与推理

### 2. 模拟器使用原则
- 不能每次都模拟100次，要有紧迫感
- 模拟中的死亡要写得有代入感
- 模拟推理过程要展现主角智慧
- 偶尔给主角意外（模拟中没遇到的情况）

### 3. 恐怖与爽感平衡
- 前期恐怖氛围重，展现怪谈的可怕
- 中期主角逐渐掌控局面，恐怖转为爽感
- 后期主角成为"怪谈中最恐怖的存在"

### 4. 黑色幽默元素
- 主角在模拟中死了无数次后的淡定
- 其他人紧张时主角的"我模拟过了"
- 怪谈们遇到主角后的"这人不对劲"

## 经典场景设计

### 场景1：模拟推理
```
【第47次模拟】
林默又一次死在了第三天。
这次是因为进入了三楼。
但第一次模拟时，他没进三楼，也死了。
"有问题。"
他眯起眼睛，开始回顾所有规则。
第三条：禁止进入三楼。
第五条：每天必须在所有楼层打卡一次。
"矛盾了。"
"所以...第三条是假规则。"
"不对，第五条也可能是假的。"
"让我再试一次。"

【第48次模拟】
这一次，他故意违反了两条规则。
然后——
他活了下来。
"原来如此。"
林默嘴角微微上扬。
"我知道真相了。"
```

```
这人...
是人吗？
```
